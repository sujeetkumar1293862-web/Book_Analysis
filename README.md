# Book_Analysis

 SQL Project: Books & Orders Analysis

1. Overview
This project showcases a comprehensive set of SQL queries designed to analyze a Books & Orders database. It covers fundamental to advanced SQL concepts including filtering, aggregation, joins, subqueries. The project demonstrates practical data analysis skills applicable to real-world scenarios such as e-commerce, inventory management, and customer analytics.

2. Features
- Retrieve and filter data based on genre, publication year, and customer location.
- Aggregate and calculate metrics like total stock, revenue, and average book prices.
- Identify top-performing books, customers, and authors.
- Advanced queries using `GROUP BY`, `HAVING`, subqueries, and aggregate functions.
- Demonstrates problem-solving skills for typical business questions:
  - Most frequently ordered books
  - Top customers by spending
  - Total books sold per genre or author
  - Orders exceeding average or maximum amounts

3. Technologies
- SQL (MySQL)
- Database concepts: JOINs, Aggregations, Subqueries

4. Project Structure
- `sales_analysis.sql` — contains all the queries, clearly labeled as Q1, Q2, etc., for easy reference.
- Queries are grouped into:
  - Basic Queries:** Filtering, sorting, simple aggregations.
  - Advanced Queries:** Complex aggregations, subqueries, top N analysis, customer insights.

5. Objective
The purpose of this project is to:
1. Demonstrate practical SQL skills.
2. Solve real-world business questions using database queries.
3. Build a portfolio-ready project to impress recruiters and data analytics hiring managers.
